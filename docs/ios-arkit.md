# iOS ARKit Capture Plan

Native iOS capture is the source of truth for guided home scans. The web app is the sharing, review, and listing presentation layer.

## Supported Capture Modes

Interior mode:

- Start an ARKit session with world tracking and LiDAR scene reconstruction where available.
- Run RoomPlan to identify walls, openings, floors, ceilings, doors, and windows.
- Show guided pass states matching the web workflow: entry alignment, walls, ceilings, floors, openings, final pass.
- Export USDZ for iOS Quick Look and glTF/GLB for the web viewer.

Exterior mode:

- Start ARKit world tracking with environment texturing.
- Sample ambient light and camera exposure to drive sunlight compensation.
- Capture facade/lot mesh where device support allows it.
- Attach Google Maps Street View coordinates and heading metadata before upload.

## Swift Module Shape

```swift
import ARKit
import RealityKit
import RoomPlan
import SwiftUI

enum ScanMode {
    case interior
    case exterior
}

struct CaptureExport {
    let usdzURL: URL
    let gltfURL: URL?
    let manifestURL: URL
}

@MainActor
final class HomeScanSession: NSObject, ObservableObject {
    @Published var mode: ScanMode = .interior
    @Published var progress: Double = 0
    @Published var trackingQuality: Double = 0

    private let arSession = ARSession()

    func start(mode: ScanMode) {
        self.mode = mode
        let configuration = ARWorldTrackingConfiguration()
        configuration.planeDetection = [.horizontal, .vertical]
        configuration.environmentTexturing = .automatic

        if ARWorldTrackingConfiguration.supportsSceneReconstruction(.mesh) {
            configuration.sceneReconstruction = .mesh
        }

        arSession.run(configuration, options: [.resetTracking, .removeExistingAnchors])
    }

    func stop() {
        arSession.pause()
    }

    func exportCapture() async throws -> CaptureExport {
        // First implementation slice:
        // 1. Export RoomPlan USDZ for interiors.
        // 2. Convert or upload a GLB generated by the server-side pipeline.
        // 3. Write a manifest with dimensions, listing metadata, capture mode, and GCS paths.
        throw CancellationError()
    }
}
```

## GCP Upload Contract

The iOS app should request a signed upload session from the API layer, then upload:

- `model.usdz`
- `model.glb` or `scene.gltf` plus textures
- `manifest.json`
- preview poster image

The web app can load the public or signed model URL with:

```text
https://your-domain.example/view?model=https%3A%2F%2Fstorage.googleapis.com%2Fbucket%2Fscan.glb
```

## First Native Milestone

1. Create an Xcode iOS app target named `HomeScanAR`.
2. Add camera, motion, photo library, and local network permission strings.
3. Implement `HomeScanSession.start(mode:)`.
4. Export a RoomPlan USDZ for a single room.
5. Upload the USDZ and manifest to Cloud Storage with a signed URL.
6. Open the returned web share URL.
